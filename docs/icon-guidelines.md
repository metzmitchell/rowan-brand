# Rowan icon guidelines

## Core style

Rowan icons should be:

- Hand-drawn
- Elegant
- Thin-line
- Simple
- Slightly organic
- Quiet, not cartoonish

They should feel like a careful sketch from a designer, not a generic icon pack.

## Line quality

- Use thin strokes
- Keep corners soft when appropriate
- Avoid heavy geometric precision
- Avoid filled pictograms unless there is a clear use case
- Avoid noisy detail at small sizes

## Color

- On light backgrounds, use a dark green tone
- On dark backgrounds, use Shell (`#f5f2e8`)
- Avoid accent-color icons unless the icon is part of a deliberate CTA moment

## Usage

Good uses:

- Small visual support next to service or process sections
- Sparse section markers
- Simple concept illustrations
- Website UI moments where text still does the main work

Avoid:

- Dense icon grids
- Cartoon mascot styles
- Generic SaaS outline sets
- Multi-color icon systems
- Icons that compete with large headings

## Prompt description

Use this when asking for icon concepts:

> Hand-drawn, elegant, thin-line icon for Rowan Builder Marketing. Minimal, calm, slightly organic linework. Use dark green on a Shell background or Shell on a dark green background. No cartoon style, no stock icon feel, no heavy fill, no bright accent colors.
